## 2. 알고리즘의 정의 및 특징 (#intro)

**알고리즘** 이란 어떤 문제를 해결하기 위해 정해진 **유한한 단계** 의 절차를 의미합니다.

### 2.1 알고리즘의 기본 구조
```text
[ Input ] (Problem)  -->  [ Algorithm ] (Finite Steps)  -->  [ Output ] (Result)
```

### 2.2 알고리즘과 자료구조
- **자료구조**: 데이터를 어떻게 **저장** 할 것인가? (예: 영어 사전의 단어 배치)
- **알고리즘**: 문제를 어떻게 **해결** 할 것인가? (예: 단어 찾기 방법)
- **핵심**: 데이터가 어떻게 조직화(정렬 등)되어 있느냐에 따라 사용할 수 있는 알고리즘의 효율성이 결정됩니다.

## 3. 역사 속의 알고리즘 (Euclid's GCD) (#history)

인류 최초의 알고리즘으로 알려진 기원전 300년경 유클리드의 **최대공약수(GCD) 알고리즘** 을 통해 알고리즘의 효율성을 이해합니다.

### 3.1 유클리드 호제법 (Euclid's Algorithm)
- **원리**: 두 수의 최대공약수는 큰 수를 작은 수로 나눈 나머지의 최대공약수와 같습니다.
- **직관적 이해**: 가로  $a$ , 세로  $b$  인 직사각형을 가장 큰 정사각형들로 빈틈없이 채울 때, 그 정사각형의 한 변의 길이가 바로 GCD입니다. 나머지를 취하는 과정은 직사각형에서 정사각형을 계속해서 잘라내는 과정과 같습니다.

```python
def gcd(a, b):
    while b != 0:
        a, b = b, a % b
    return a
```

## 4. 탐색 알고리즘 (Search Algorithms) (#search)

### 4.1 순차 탐색 vs 이진 탐색
| 방식 | 원리 | 최악의 경우 ( $n$  개) | 조건 |
| :--- | :--- | :--- | :--- |
| **순차 탐색** | 처음부터 끝까지 하나씩 확인 |  $n$  번 | 없음 |
| **이진 탐색** | 중간값을 기준으로 범위를 절반씩 축소 |  $\log\_{2}{n}$  번 | **정렬 필수** |

### 4.2 이진 탐색 (Binary Search) 작동 원리
정렬된 배열에서 타겟  **7**  을 찾는 과정:
1.  **배열**: `[1, 3, 5, 7, 9, 11, 13]`
2.  **Step 1**: 중간값 `7` 확인. **발견!** (단 1번 만에 탐색 완료)

만약 타겟이  **3**  이라면:
1.  **Step 1**: 중간값 `7`과 비교.  $3 \lt 7$  이므로 왼쪽 범위 `[1, 3, 5]` 선택.
2.  **Step 2**: 중간값 `3` 확인. **발견!** (7개 데이터 중 단 2번 만에 완료)

## 5. 탐욕 알고리즘 (Greedy Algorithm) (#greedy)

매 순간 **현 시점에서 가장 최선인 선택** 을 내리는 전략입니다.

### 5.1 동전 교환 문제 (Coin Change)
- **전략**: 가장 큰 단위의 동전부터 최대한 많이 사용합니다.
- **성공 사례**: {500, 100, 50, 10} 단위. 큰 단위가 작은 단위의 배수이므로 항상 최적해를 보장합니다.
- **실패 사례 (맛보기)**: 동전 단위가 {500, 400, 100}이고 거스름돈이 800원인 경우.
    - 그리디: 500 + 100 + 100 + 100 (4개)
    - **최적해**: 400 + 400 (2개) -> 이 경우 **동적 계획법 (DP)** 이 필요합니다.

## 6. 고전적 문제 해결 (Euler & Maze) (#classic)

### 6.1 오일러 경로 (Euler Path)
모든 간선을 정확히 한 번씩만 지나는 경로입니다. (한 붓 그리기)
- **핵심 전략**: 선택지가 있을 때, **브리지 (Bridge, 끊으면 그래프가 분리되는 간선)** 를 피하고 **사이클 (Cycle)** 이 있는 쪽을 먼저 택합니다.

### 6.2 미로 찾기 (Right-Hand Rule)
벽에 오른손을 대고 계속 따라가면 반드시 출구에 도달한다는 알고리즘입니다. 벽이 하나로 연결되어 있다는 기하학적 특징을 이용한 단순하고 우아한 해결책입니다.

## 7. 효율적인 문제 해결 (Divide & Conquer) (#divide)

문제를 더 작은 단위로 나누어 해결함으로써 기하급수적인 성능 향상을 얻는 방식입니다.

### 7.1 가짜 동전 찾기 (Fake Coin)
$n$  개의 동전 중 아주 약간 가벼운 가짜 동전 1개를 찾아야 합니다.

- **그리디(하나씩 비교)**: 하나를 빼두고 나머지를 하나씩 저울에 답니다. 운이 없으면  $n-1$  번 확인해야 합니다.
- **분할 정복(절반씩 비교)**: 동전을 두 무리로 나누어 저울에 답니다. 가벼운 쪽 무리를 다시 절반으로 나눕니다.

**[이진 분할 과정 시각화 (동전 8개)]**
```text
Step 1: [1,2,3,4] vs [5,6,7,8]  -->  [1,2,3,4] 가 가벼움
Step 2: [1,2] vs [3,4]          -->  [1,2] 가 가벼움
Step 3: [1] vs [2]              -->  [1] 이 가벼움 (발견!)
```
- **결과**: 단 3번 만에 해결. 일반화하면  $\log\_{2}{n}$  번의 비교만으로 충분합니다.

## 8. 이진 인코딩 (Poisoned Wine) (#binary)

1,000개의 포도주 병 중 독이 든 1병을 찾아야 합니다. 독은 1주일 뒤에 반응하며, 최소 인원으로 가장 빨리 찾는 방법은 무엇일까요?

### 8.1 비트 인코딩 전략 (Bit Encoding)
각 포도주 병에 0부터 999까지 **이진수 번호** 를 부여합니다. (1,000은  $2^{10}=1024$  보다 작으므로 10비트면 충분합니다.)

**[신하와 비트의 대응]**
- 10명의 신하를 각 비트 자리수( $2^{0}$  ~  $2^{9}$ )에 대응시킵니다.
- 각 신하는 자신의 담당 비트가 **1**인 모든 포도주를 한 모금씩 마십니다.

**[비트 테이블 예시]**
| 병 번호 | 이진수 (10비트) | 마시는 신하 (1이면 마심) |
| :--- | :--- | :--- |
| **병 #1** | `0000000001` | 신하 #1 |
| **병 #3** | `0000000011` | 신하 #1, 신하 #2 |
| **병 #5** | `0000000101` | 신하 #1, 신하 #3 |

### 8.2 결과 도출
1주일 뒤, **죽은 신하들이 있는 자리를 1, 살아남은 자리를 0**으로 치환하여 이진수를 읽습니다.
- **예시**: 신하 #1, #3이 죽고 나머지가 살았다면? -> `0000000101` (이진수) = **5번 병** 이 독포도주!
- **시사점**: 복잡한 정보도 **이진수(Binary)** 로 인코딩하면 최소한의 자원(10명)으로 효율적인 검색이 가능합니다.
