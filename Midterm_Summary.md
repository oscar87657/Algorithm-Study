## 1. 알고리즘의 본질과 로그의 마법 (W01)

<a id="intro_1"></a>
### 1.1. 알고리즘의 정의와 3대 핵심 성질

- **정의**: 입력(문제)으로부터 **유한한 단계** 내에 **정확한 출력**(결과)을 내는 **명확한 절차**.
- **알고리즘 vs 자료구조**: `알고리즘 + 자료구조 = 프로그램` (Niklaus Wirth).
- **좋은 알고리즘의 3대 조건**:
    1. **유한성 (Finiteness)**: 반드시 유한한 시간/단계 내에 종료되어야 함.
    2. **정확성 (Correctness)**: 모든 유효한 입력에 대해 항상 올바른 결과(출력)를 도출해야 함.
    3. **명확성 (Definiteness/Unambiguity)**: 각 단계는 모호함 없이 수행 가능하도록 명확히 정의되어야 함. (예: "적당히 처리한다" (X))

<a id="intro_2"></a>
### 1.2. 로그($\log$) 복잡도의 위력과 계산

- **직관적 의미**: "$n$ 을 $1$ 이 될 때까지 $2$ 로 몇 번 나눌 수 있는가?"
- **순차 탐색 vs 이진 탐색**: 데이터 $n = 1,000,000$ 개일 때, 순차 탐색은 $1,000,000$ 번, 이진 탐색은 단 **$20$ 번** ($\log\_{2}{10^{6}} \approx 20$).
- **성능 특징**: 입력 크기가 기하급수적으로 늘어나도 연산 횟수는 완만하게 증가 ($O(\log n)$).

<a id="intro_3"></a>
### 1.3. 고전적 문제 해결 전략 (시험 단골)

- **가짜 동전 찾기 (양팔 저울)**:
    - **2분할 전략**: 매 측정마다 후보를 절반으로 축소 $\rightarrow \lceil \log\_{2}{n} \rceil$ 번.
    - **3분할 전략 (최적)**: 세 더미로 나눠 두 더미만 비교 $\rightarrow \lceil \log\_{3}{n} \rceil$ 번. (예: $1,024$ 개 동전 $\rightarrow 7$ 번 ($3^{7}=2187$)).
- **독이 든 와인 문제 (이진 인코딩)**:
    - **핵심**: 하인의 상태(생존/사망)를 비트($0/1$)로 활용하여 항아리 번호를 인코딩.
    - **필요 하인 수**: $\lceil \log\_{2}{n} \rceil$ 명. (예: $1,000$ 개 항아리 $\rightarrow 10$ 명 ($2^{10}=1024$)).
    - **방법**: $k$ 번째 하인은 항아리 번호를 이진수로 바꿨을 때 $k$ 번째 비트가 $1$ 인 와인을 모두 시음.
- **오일러 경로 (한붓그리기)**:
    - **조건**: 모든 간선을 정확히 한 번씩 통과.
    - **오일러 순환 (Circuit)**: 시작점 = 도착점. 모든 정점의 **차수(Degree)가 짝수**.
    - **오일러 경로 (Path)**: 시작점 $\ne$ 도착점. 홀수 차수 정점이 정확히 **$2$ 개**.
    - **전략**: **'다리(Bridge)'** (건너면 그래프가 끊기는 간선)는 사이클이 없는 경우에만 마지막에 건넘.
- **유클리드 알고리즘 (GCD)**:
    - **원리**: $gcd(a, b) = gcd(b, a \pmod{b})$.
    - **특징**: 단순 반복으로 최대공약수를 구하는 가장 효율적인 방법.

<a id="intro_4"></a>
### 1.4. 알고리즘 설계의 핵심 연결고리

- **패턴**: `문제 해결` $\rightarrow$ `분할 정복` $\rightarrow$ `재귀적 사고` $\rightarrow$ `점화식`.
- **예시**: 이진 탐색($T(n) = T(n/2) + O(1)$), 가짜 동전 찾기 등.

## 2. 알고리즘 설계 및 복잡도 분석 (W02)

<a id="complexity_1"></a>
### 2.1. 점근적 표기법의 수학적 정의

- **Big-O ($O$)**: 상한. $f(n) \le c \cdot g(n) \quad (n \ge n\_{0})$.
- **Big-Omega ($\Omega$)**: 하한. $f(n) \ge c \cdot g(n) \quad (n \ge n\_{0})$.
- **Big-Theta ($\Theta$)**: Tight bound. $c\_{1} \cdot g(n) \le f(n) \le c\_{2} \cdot g(n)$.

<a id="complexity_2"></a>
### 2.2. 시간 복잡도 클래스 비교

$O(1) \lt O(\log n) \lt O(n) \lt O(n \log n) \lt O(n^{2}) \lt O(2^{n})$.

<a id="complexity_3"></a>
### 2.3. 효율성 분석 기법 (루프 및 재귀)

- **루프**: 중첩 횟수에 따라 $O(n^{k})$.
- **재귀 트리**: 각 호출의 비용을 시각화하여 전체 합산.

<a id="complexity_4"></a>
### 2.4. 최선, 평균, 최악의 경우 분석

- **최악의 경우 (Worst-case)**: 시스템 안정성을 위해 가장 중요하게 고려함 ($O$ 표기).

## 3. 기본 자료구조 및 정렬 알고리즘 (W03)

<a id="sorting_1"></a>
### 3.1. 선형 자료구조 (Linear Data Structures)

- **배열**: 임의 접근 $O(1)$, 중간 삽입/삭제 $O(n)$.
- **스택 (LIFO)**: `push`, `pop`, `peek`. (함수 호출, Undo).
- **큐 (FIFO)**: `enqueue`, `dequeue`. (스케줄링, BFS).

<a id="sorting_2"></a>
### 3.2. 기초 정렬 알고리즘 (Basic Sorting Algorithms)

- **선택 정렬**: 최솟값 찾아 교환. 항상 $O(n^{2})$. **불안정 정렬**.
- **삽입 정렬**: 정렬된 부분에 삽입. 최악 $O(n^{2})$, 최선(정렬됨) $O(n)$. **안정 정렬**.
- **버블 정렬**: 인접 원소 비교/교환. 평균/최악 $O(n^{2})$. **안정 정렬**.

<a id="sorting_3"></a>
### 3.3. 정렬 알고리즘의 심화 개념

- **안정성 (Stability)**: 동일 키 순서 유지 여부.
- **제자리 정렬 (In-place)**: 추가 공간 $O(1)$ 사용.

## 4. 분할 정복 알고리즘 (W04)

<a id="divide_and_conquer_1"></a>
### 4.1. 분할 정복의 설계 전략 (3단계)

1. **Divide** (분할) $\rightarrow$ 2. **Conquer** (정복/재귀) $\rightarrow$ 3. **Combine** (결합).

<a id="divide_and_conquer_2"></a>
### 4.2. 주요 분할 정복 정렬 알고리즘

- **합병 정렬 (Merge Sort)**: 항상 $O(n \log n)$. **안정 정렬**. 추가 공간 $O(n)$ 필요.
- **퀵 정렬 (Quick Sort)**: 평균 $O(n \log n)$, 최악 $O(n^{2})$. **제자리 정렬**. 피벗 선택이 성능 좌우.

<a id="divide_and_conquer_3"></a>
### 4.3. 재귀식과 알고리즘 분석 도구

- **마스터 정리 (Master Theorem)**: $T(n) = aT(n/b) + f(n)$ 해결 공식.
- **이진 탐색**: $T(n) = T(n/2) + O(1) \rightarrow O(\log n)$.

## 5. 탐욕 알고리즘 (W05)

<a id="greedy_1"></a>
### 5.1. 탐욕 알고리즘의 기초 및 성립 조건

- **탐욕적 선택 속성**: 지역 최적이 전역 최적으로 이어짐.
- **최적 부분 구조**: 문제의 최적해가 부분 문제의 최적해를 포함함.

<a id="greedy_2"></a>
### 5.2. 활동 선택 문제 (Activity Selection)

- **핵심 전략**: **'종료 시간' ($f\_{i}$)** 이 가장 빠른 것부터 선택.

<a id="greedy_3"></a>
### 5.3. 허프만 코딩 (Huffman Coding)

- **원리**: 빈도 낮은 두 노드를 합쳐 트리 구축 (최소 힙 사용). **가변 길이 접두어 코드**.

<a id="greedy_4"></a>
### 5.4. 분할 가능 배낭 문제 (Fractional Knapsack)

- **전략**: **'단위 무게당 가치'** 가 높은 순서대로 담기. (그리디 가능)

## 6. 동적 계획법 (DP) (W06)

<a id="dp_1"></a>
### 6.1. 동적 계획법의 성립 조건

- **최적 부분 구조** + **중복되는 부분 문제 (Overlapping Subproblems)**.

<a id="dp_2"></a>
### 6.2. 메모이제이션 (Top-down) vs 타뷸레이션 (Bottom-up)

- **Memoization**: 재귀 + 결과 저장. 필요한 것만 계산.
- **Tabulation**: 반복문 + 표 채우기. 오버헤드 적고 안정적.

<a id="dp_3"></a>
### 6.3. 최장 공통 부분 수열 (LCS)

- **점화식**:
    - 같으면: $c[i, j] = c[i-1, j-1] + 1$
    - 다르면: $c[i, j] = \max(c[i-1, j], c[i, j-1])$

<a id="dp_4"></a>
### 6.4. 0-1 배낭 문제 (0-1 Knapsack Problem)

- **점화식**: $dp[i][w] = \max(dp[i-1][w], v\_{i} + dp[i-1][w - w\_{i}])$
- 물건을 쪼갤 수 없으므로 그리디 불가, DP 필수.
