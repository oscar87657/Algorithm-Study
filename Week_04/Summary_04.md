<a id="concept"></a>
## 1. 분할 정복이란? (#concept)

어려운 문제를 만났을 때 당황하지 말고 딱 세 가지만 기억하세요.

1. **분할 (Divide)**: 문제를 똑같은 모양의 작은 하위 문제들로 나눕니다.
2. **정복 (Conquer)**: 작아진 문제들을 하나씩 해결합니다. (충분히 작다면 바로 풀기!)
3. **결합 (Combine)**: 풀린 작은 답들을 다시 합쳐서 원래 문제의 정답을 만듭니다.

> **멘토의 한마디**: 분할 정복의 마법은 **"문제가 반으로 줄어들 때"** 일어납니다. 반씩 줄어드는 작업은 아무리 양이 많아도 로그( $\log n$ ) 시간 안에 끝낼 수 있기 때문입니다.

<a id="binary-search"></a>
## 2. 이진 탐색 (Binary Search) (#binary-search)

정렬된 데이터에서 범위를 **절반씩 버리며** 타겟을 찾는 가장 대표적인 사례입니다.

- **핵심 아이디어**: "가운데 값보다 작네? 그럼 오른쪽 절반은 볼 필요도 없지!"
- **복잡도**: $\Theta(\log n)$ (데이터가 100만 개여도 20번이면 끝납니다.)

```python
def binary_search(arr, low, high, target):
    if low > high: return -1 # 찾을 수 없음
    
    mid = (low + high) // 2
    if arr[mid] == target: return mid # 정답!
    
    # 정복: 한쪽 반은 과감히 버리고, 나머지 반만 탐색
    if arr[mid] > target:
        return binary_search(arr, low, mid - 1, target)
    else:
        return binary_search(arr, mid + 1, high, target)
```

<a id="selection"></a>
## 3. 선택 문제 (Selection Problem) (#selection)

$n$ 개의 숫자 중 **$k$ 번째로 작은 수** 를 찾는 문제입니다. 전체를 다 정렬($n \log n$)할 필요 없이 필요한 부분만 파고듭니다.

### 3.1 무작위 선택 (Randomized Select)
피벗(기준점)을 하나 잡아서 그보다 작은 그룹, 큰 그룹으로 나눕니다.
- **포인트**: 내가 찾는 $k$ 번째가 있는 쪽 한 군데만 골라 들어갑니다. (평균적으로 선형 시간 $\Theta(n)$ 소요)

### 3.2 Median of Medians (MoM): 최악을 방지하는 전략
피벗이 운 나쁘게 항상 맨 끝값으로 잡히면 속도가 느려집니다. 이를 방지하기 위해 **"5명씩 팀을 짜서 대표(중앙값)를 뽑는"** 전략을 씁니다.
- **왜 5명인가?**: 수학적으로 너무 적지도, 많지도 않게 피벗의 퀄리티(상위/하위 30% 보장)를 유지해주는 최소한의 숫자입니다.
- **결론**: 이 꼼꼼한 피벗 선택 덕분에 어떤 최악의 상황에서도 **$\Theta(n)$** 속도를 보장합니다.

<a id="closest-pair"></a>
## 4. 최근접 점의 쌍 (Closest Pair) (#closest-pair)

평면 위에서 가장 가까운 두 점을 찾는 문제입니다.

- **문제점**: 모든 점을 짝지어 비교하면 너무 오래 걸립니다 ($n^{2}$ 번).
- **해결책**:
    1. 왼쪽/오른쪽 영역을 나눠서 각각의 최단 거리 $d$ 를 구합니다.
    2. **기하학적 통찰**: 이제 "왼쪽 하나, 오른쪽 하나" 걸쳐있는 점들만 보면 되는데, 경계선 기준 좌우로 $d$ 만큼의 띠 영역만 조사하면 충분합니다.
    3. **진짜 꼼수**: 띠 안의 한 점을 기준으로, **주변 6개 정도의 점** 만 확인하면 됩니다. (다른 점들은 거리가 $d$ 보다 클 수밖에 없기 때문!)

<a id="limitation"></a>
## 5. 주의사항: 분할 정복을 쓰면 안 되는 경우 (#limitation)

### 5.1 피보나치 수열 (Fibonacci)
분할 정복은 "문제를 쪼개서 각자 풀 때" 효율적입니다. 하지만 피보나치처럼 **똑같은 문제를 여러 번 반복해서 쪼개야 하는 경우**는 치명적입니다.

- **문제**: $F(5)$ 를 구하려고 $F(4), F(3)$ 으로 쪼갰는데, $F(4)$ 를 풀 때 $F(3)$ 을 또 풀어야 합니다.
- **결론**: 하위 문제가 서로 겹칠 때는 분할 정복 대신 **동적 계획법 (DP)** 을 사용하여 한 번 푼 문제는 저장해두고 꺼내 써야 합니다.
