<a id="fundamentals"></a>
## 1. 알고리즘의 기초 및 특성 (#fundamentals)

알고리즘은 문제를 해결하기 위한 **유한한 단계** 의 절차입니다. 좋은 알고리즘은 다음 4가지 특성을 가져야 합니다.

| 특성 | 설명 |
| :--- | :--- |
| **정확성 (Correctness)** | 모든 유효한 입력에 대해 항상 정확한 결과를 출력해야 함. |
| **수행 가능성 (Executability)** | 각 단계는 컴퓨터에서 실제로 실행 가능해야 함. |
| **유한성 (Finiteness)** | 알고리즘은 반드시 유한한 시간 내에 종료되어야 함. |
| **효율성 (Efficiency)** | 시간과 메모리 자원을 적게 사용할수록 가치가 높음. |

<a id="representation"></a>
## 2. 알고리즘의 표현과 재귀 (Recursion) (#representation)

### 2.1 표현 방법
- **자연어 (Natural Language)**: 일반 언어 설명 (이해는 쉬우나 모호함).
- **의사코드 (Pseudocode)**: 구조적이고 명확한 표기법 (표준적 방법).
- **순서도 (Flowchart)**: 도형을 이용한 시각적 도식화 (단순한 로직에 유리).

### 2.2 재귀와 호출 스택 (Call Stack)
재귀 함수는 자기 자신을 호출할 때마다 메모리의 **스택 (Stack)** 영역에 새로운 프레임을 쌓습니다(Push). 기저 조건에 도달하면 결과값이 반환되며 스택에서 사라집니다(Pop).

**예시: 유클리드 GCD 재귀 호출 과정 (24, 14)**
```text
gcd_rec(24, 14)
  └─> gcd_rec(14, 10)
        └─> gcd_rec(10, 4)
              └─> gcd_rec(4, 2)
                    └─> gcd_rec(2, 0)  -->  결과 2 반환 (Pop 시작)
```

## 3. 알고리즘 설계 전략 분류 (#classification)

| 유형 | 핵심 아이디어 | 대표 사례 |
| :--- | :--- | :--- |
| **분할 정복 (Divide & Conquer)** | 큰 문제를 독립된 작은 문제로 나누어 해결 및 결합 | 병합 정렬, 이진 탐색 |
| **탐욕법 (Greedy)** | 매 순간 국부적인 최적해를 선택 | 동전 교환, 크루스칼 |
| **동적 계획법 (DP)** | 하위 문제의 해를 저장하고 재사용 (중복 문제 해결) | 피보나치, 배낭 문제 |
| **근사 알고리즘 (Approximation)** | NP-hard 문제에 대해 최적에 가까운 해를 구함 | TSP 근사해 |
| **백트래킹 (Backtracking)** | 모든 가능성을 탐색하되, 안 되는 길은 즉시 포기 (Pruning) | N-Queens, 미로 찾기 |
| **분기 한정 (Branch & Bound)** | 백트래킹에 상한/하한을 적용하여 탐색 범위 축소 | TSP 최적해 |

<a id="complexity"></a>
## 4. 시간 복잡도와 사례 분석 (#complexity)

### 4.1 복잡도 분석의 종류 (출퇴근 비유)
- **최악 경우 (Worst-case)**: 지하철을 바로 앞에서 놓쳐서 최대 대기 시간이 포함된 경우. (**알고리즘 분석의 표준**)
- **평균 경우 (Average-case)**: 일반적인 대기 시간을 포함한 평균적인 통학 시간.
- **최선 경우 (Best-case)**: 도착하자마자 지하철이 바로 들어오는 행운의 경우.

### 4.2 복잡도 클래스별 코드 예시 (Python)

**O(1) - 상수 시간**: 입력 크기 $n$ 에 관계없이 항상 일정함.
```python
def sample1(A, n):
    k = n // 2
    return A[k] # 단 한 번의 연산
```

**O(n) - 선형 시간**: 데이터 개수에 비례하여 시간이 늘어남.
```python
def sample2(A, n):
    sum = 0
    for i in range(n):
        sum += A[i] # n번 반복
    return sum
```

**O(n^2) - 이차 시간**: 중첩 반복문이 있는 경우.
```python
def sample3(A, n):
    sum = 0
    for i in range(n):
        for j in range(n):
            sum += A[i] * A[j] # n * n번 반복
    return sum
```

## 5. 점근적 표기법 (Asymptotic Notation) (#asymptotic)

데이터 크기 $n$ 이 아주 커질 때, 사소한 항들을 버리고 **가장 영향력이 큰 항** 만 남겨 표현하는 방식입니다.

### 5.1 주요 표기법
1. **Big-O ( $O$ )**: 점근적 상한. "아무리 느려도 이보다는 빠르다."
2. **Big-Omega ( $\Omega$ )**: 점근적 하한. "아무리 빨라도 이보다는 느리다."
3. **Theta ( $\Theta$ )**: 동일한 증가율. 상한과 하한이 일치하는 정확한 성능.

### 5.2 수학적 증명 예시
**문제**: $f(n) = 2n^{2} - 8n + 3 = O(n^{2})$ 임을 증명하라.
- **증명**: 정의에 따라 $2n^{2} - 8n + 3 \le c \cdot n^{2}$ 를 만족하는 상수 $c$ 와 $n\_{0}$ 를 찾아야 합니다.
- 만약 $c = 5$ 라면, $2n^{2} - 8n + 3 \le 5n^{2}$ 는 $-3n^{2} - 8n + 3 \le 0$ 이 되며, 이는 $n \ge 1$ 일 때 항상 참입니다.
- 따라서 $c = 5, n\_{0} = 1$ 로 두면 $f(n) = O(n^{2})$ 입니다.

<a id="recurrence"></a>
## 6. 재귀식 풀이법 (Recurrence Relations) (#recurrence)

재귀 알고리즘의 실행 시간을 구하는 세 가지 주요 방법입니다.

### 6.1 반복 대입법 (Repeated Substitution)
식에 계속해서 식을 대입하여 패턴을 찾는 방법입니다.
- **예시**: $T(n) = T(n-1) + c, T(1) = c$
- $T(n) = (T(n-2) + c) + c = T(n-2) + 2c = \dots = T(1) + (n-1)c$
- 결과: $T(n) = cn = O(n)$ (팩토리얼의 복잡도)

### 6.2 추측후 증명법 (Guess and Verify)
답을 미리 짐작하고 수학적 귀납법으로 확인하는 방법입니다.
- **주의**: 추측한 식의 상수 $c$ 가 증명 과정에서도 변하지 않고 유지되어야 함을 확인해야 합니다.

## 7. 마스터 정리 (Master Theorem) - 쉬운 풀이 (#master)

$T(n) = aT(n/b) + f(n)$ 형태의 점화식을 한 방에 푸는 공식입니다.
( $a$: 하위 문제 개수, $b$: 문제 감소 비율, $f(n)$: 병합 비용)

**핵심 전략: "누가 더 힘이 센가?"**
하위 문제를 푸는 비용( $h(n) = n^{\log\_{b}{a}}$ )과 병합 비용( $f(n)$ )을 비교합니다.

| 상황 | 승자 (Winner) | 결과 복잡도 ( $T(n)$ ) |
| :--- | :--- | :--- |
| **Case 1**: 하위 문제가 훨씬 무거울 때 | 하위 문제 승 | $\Theta(n^{\log\_{b}{a}})$ |
| **Case 2**: 두 비용이 비등할 때 | 무승부 (둘 다 중요) | $\Theta(n^{\log\_{b}{a}} \log\_{2}{n})$ |
| **Case 3**: 병합 비용이 훨씬 무거울 때 | 병합 비용 승 | $\Theta(f(n))$ |

**예시 풀이:**
1. $T(n) = 2T(n/2) + n$
   - $a=2, b=2 \implies h(n) = n^{\log\_{2}{2}} = n^{1} = n$
   - $f(n) = n \implies h(n)$ 과 $f(n)$ 이 같음 (**Case 2**)
   - 결과: $T(n) = \Theta(n \log\_{2}{n})$ (병합 정렬)

2. $T(n) = 2T(n/4) + n$
   - $a=2, b=4 \implies h(n) = n^{\log\_{4}{2}} = n^{0.5} = \sqrt{n}$
   - $f(n) = n \implies f(n)$ 이 훨씬 큼 (**Case 3**)
   - 결과: $T(n) = \Theta(n)$

## 💡 요약: 왜 알고리즘인가?
하드웨어의 발전보다 알고리즘의 개선이 성능 향상에 더 큰 기여를 합니다.
- 10억 개의 데이터를 정렬할 때, $O(n^{2})$ 알고리즘은 슈퍼컴퓨터로 **1주일** 이 걸리지만, $O(n \log\_{2}{n})$ 알고리즘은 개인 PC에서 단 **5분** 이면 충분합니다.
- **효율적인 알고리즘은 슈퍼컴퓨터보다 강력합니다.**
