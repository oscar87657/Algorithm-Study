<a id="fundamentals"></a>
## 1. 알고리즘의 기초 및 특성

알고리즘은 문제를 해결하기 위한 **유한한 단계** 의 절차입니다.

| 특성 | 설명 |
| :--- | :--- |
| **정확성 (Correctness)** | 모든 유효한 입력에 대해 항상 정확한 결과를 출력해야 함. |
| **수행 가능성 (Executability)** | 각 단계는 컴퓨터에서 실제로 실행 가능해야 함. |
| **유한성 (Finiteness)** | 알고리즘은 반드시 유한한 시간 내에 종료되어야 함. |
| **효율성 (Efficiency)** | 시간과 메모리 자원을 적게 사용할수록 가치가 높음. |

## 2. 알고리즘의 표현과 재귀 (Recursion)

### 2.1 재귀와 호출 스택 (Call Stack)
재귀 함수는 자기 자신을 호출할 때마다 메모리의 **스택 (Stack)** 영역에 새로운 프레임을 쌓습니다.

**예시: 유클리드 GCD 재귀 호출 과정 (24, 14)**
```text
gcd(24, 14)
  └─> gcd(14, 10)  (24 % 14 = 10)
        └─> gcd(10, 4)  (14 % 10 = 4)
              └─> gcd(4, 2)  (10 % 4 = 2)
                    └─> gcd(2, 0)  --> 결과 2 반환 (Pop 시작)
```

## 3. 알고리즘 설계 전략 분류

| 유형 | 핵심 아이디어 | 대표 사례 |
| :--- | :--- | :--- |
| **분할 정복 (Divide & Conquer)** | 큰 문제를 독립된 작은 문제로 나누어 해결 | 병합 정렬, 이진 탐색 |
| **탐욕법 (Greedy)** | 매 순간 국부적인 최적해를 선택 | 동전 교환, 크루스칼 |
| **동적 계획법 (DP)** | 하위 문제의 해를 저장하고 재사용 | 피보나치, 배낭 문제 |
| **백트래킹 (Backtracking)** | 모든 가능성을 탐색하되 가지치기(Pruning) 수행 | N-Queens, 미로 찾기 |

## 4. 시간 복잡도와 성능 비교

알고리즘의 성능은 입력 크기  $n$  이 커짐에 따라 연산 횟수가 어떻게 증가하는지로 결정됩니다.

### 4.1 복잡도 클래스 비교 그래프 (성능 차이)
```text
연산 횟수 (Time)
  ^
  |          / O(2^n) - 지수 시간 (위험!)
  |         /
  |        /   / O(n²) - 이차 시간
  |       /   /
  |      /   /   / O(n log n) - 효율적 정렬
  |     /   /   /
  |    /   /   /   / O(n) - 선형 시간
  |   /   /   /   /
  |  /   /   /   /   / O(log n) - 로그 시간
  +--------------------------------------> 데이터 크기 (n)
```

### 4.2 데이터 크기에 따른 실행 시간 예시 (1초당 1억 번 연산 기준)
|  $n$  |  $\log\_{2}{n}$  |  $n$  |  $n \log\_{2}{n}$  |  $n^{2}$  |  $2^{n}$  |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **10** | 3.3 | 10 | 33 | 100 | 1,024 |
| **100** | 6.6 | 100 | 664 | 10,000 |  $1.2 \times 10^{30}$  |
| **1,000** | 10 | 1,000 | 10,000 | 1,000,000 | **측정 불가** |

<a id="asymptotic"></a>
## 5. 점근적 표기법 (Asymptotic Notation)

데이터 크기가 무한히 커질 때 알고리즘의 대략적인 성능 추세를 나타냅니다.

### 5.1 시각적 이해
```text
성능(비용)
  ^
  |      / f(n) = O(g(n))  <-- 상한 (Upper Bound)
  |     /
  |    /--- 알고리즘 실제 성능
  |   /
  |  / f(n) = Ω(g(n))  <-- 하한 (Lower Bound)
  +---------------------------> n
```

1.  **Big-O ( $O$ )**: "아무리 느려도 이보다는 빠르다." (최악의 경우 보장)
2.  **Big-Omega ( $\Omega$ )**: "아무리 빨라도 이보다는 느리다." (최선의 성능 한계)
3.  **Theta ( $\Theta$ )**: "정확히 이 정도 속도로 증가한다." (상한과 하한이 일치)

<a id="recurrence"></a>
## 6. 재귀식 풀이법 (Recurrence Relations)

### 6.1 반복 대입법 (Repeated Substitution)
- **예시**:  $T(n) = T(n-1) + n, T(1) = 1$  (1부터  $n$  까지의 합)
-  $T(n) = (T(n-2) + (n-1)) + n = \dots = 1 + 2 + \dots + n$ 
- 결과:  $T(n) = \frac{n(n+1)}{2} = \Theta(n^{2})$

<a id="master"></a>
## 7. 마스터 정리 (Master Theorem)

재귀 관계식  $T(n) = aT(n/b) + f(n)$  을 해결하는 가장 빠른 도구입니다.

**[비교 대상]**
-  $h(n) = n^{\log\_{b}{a}}$  : 하위 문제를 해결하는 총 비용 (Leaf 비용)
-  $f(n)$  : 문제를 나누고 합치는 비용 (Divide/Combine 비용)

**[세 가지 케이스 요약]**

| 경우 | 관계 (Winner) | 시간 복잡도  $T(n)$  | 대표 예시 |
| :--- | :--- | :--- | :--- |
| **Case 1** |  $h(n) \gg f(n)$  |  $\Theta(n^{\log\_{b}{a}})$  | 피보나치 (단순 재귀) |
| **Case 2** |  $h(n) \approx f(n)$  |  $\Theta(n^{\log\_{b}{a}} \log\_{2}{n})$  | **합병 정렬, 이진 탐색** |
| **Case 3** |  $h(n) \ll f(n)$  |  $\Theta(f(n))$  | 퀵 정렬 (최선) |

## 💡 멘토의 팁: 왜 알고리즘인가?
하드웨어의 발전(슈퍼컴퓨터)보다 **알고리즘의 효율성 개선** 이 성능 향상에 더 큰 기여를 합니다. 효율적인 알고리즘은 자원이 한정된 환경에서 빛을 발하며, 복잡한 시스템의 핵심 경쟁력이 됩니다.
